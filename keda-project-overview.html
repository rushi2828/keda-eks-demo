Key Files Explained
⚡
ScaledObject
manifests/keda/scaledobject.yaml
The heart of the demo. Links your Consumer Deployment to SQS. Configures minReplicas=0 (scale-to-zero), maxReplicas=20, and 1 pod per 5 messages.
keda.sh/v1alpha1
🔐
TriggerAuthentication
manifests/keda/triggerauthentication.yaml
Uses AWS Pod Identity (IRSA) so KEDA can call SQS APIs without storing any credentials. Fully IAM-native.
IRSA · No Secrets
🐍
Consumer Worker
src/consumer/main.py
Long-polling SQS consumer. Receives up to 10 messages at once, processes them, and deletes on success. Runs in KEDA-managed pods.
Python · boto3
🏗️
Cluster Bootstrap
scripts/setup-cluster.sh
Creates EKS cluster via eksctl with OIDC enabled (required for IRSA). Managed node group with auto-scaling 1–5 nodes.
eksctl · OIDC
🔄
CI Pipeline
.github/workflows/ci.yml
Runs flake8 linting, unit tests, validates K8s manifests, builds Docker images, and pushes to ECR on every merge to main.
GitHub Actions · ECR
📨
Demo Load Script
scripts/demo-load.sh
Sends N messages to SQS to trigger KEDA scaling. Shows queue depth in real-time. Watch pods appear with kubectl get pods -w.
Demo · Load Test
